Web UI Improvement Ideas

Select the areas you'd like to explore. Multi-select is fine.

A

Richer Now Playing

Show track title, artist, and progress from HA media player attributes. The entity has this data - we're just not using it. Turn the cover section into a proper now-playing display.

B

Live Matrix Preview

Render a tiny 64x64 pixel preview of what's actually on the LED matrix right now - including screensaver animations. WebSocket already pushes state; we'd add pixel data.

C

Palette Color Swatches

Show the actual colors of each palette as gradient swatches next to the dropdown, so you can see what "Aurora" or "Ember" looks like before selecting.

D

Auto-Save Settings

Remove the explicit Save button. Apply changes immediately as you adjust them (brightness already works this way on the backend). Debounce rapid changes. Show a subtle "saved" indicator.

E

CO2 Status in UI

The data is already being sent via WebSocket but not displayed. Show current CO2 value with a color indicator (green/amber/red) in the header area near the status badge.

F

Album-Adaptive Accent Color

When music is playing, tint the web UI's accent color to match the dominant color of the album art. The WLED color extraction already does this - reuse it for the UI.
